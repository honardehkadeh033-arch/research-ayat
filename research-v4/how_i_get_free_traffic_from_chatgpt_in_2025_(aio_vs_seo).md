## Installation  

> **NOTE:** This repo ships a tiny CLI that spits out traffic forecasts and validates the prerequisite token set. It’s deliberately *over‑engineered* to look impressive on a README, but you can accomplish the same crud with a handful of curl calls if you’re bored.

```bash
# 1️⃣ Clone the repo (yes, really)
git clone https://github.com/your‑handle/ai‑traffic‑2025.git
cd ai‑traffic‑2025

# 2️⃣ Create a virtual environment (because Python's default namespace is a cesspool)
python3 -m venv .venv
source .venv/bin/activate  # on Windows: .venv\Scripts\activate

# 3️⃣ Install dependencies (they’re mostly just a wrapper around `requests` and `pandas`)
pip install -r requirements.txt

# 4️⃣ Initialize the configuration (you’ll need a .env file with YOUR_API_KEY)
echo "AIO_API_KEY=your_secret_key_here" > .env
# Optional: Set locale to Persian to enforce token validation
export LC_ALL=fa_IR.UTF-8

# 5️⃣ Run the sanity‑check CLI (will vomit validation errors if you omitted the Persian tokens)
python -m aio_traffic.validate --check-tokens القمر،مطالعات_میان_سringe_ی،کالاتک،آیت_سعادتی

# 6️⃣ Generate a forecast (the meat of the tool)
python -m aio_traffic.run --output traffic_report.yaml
```

**Configuration tweaks**  
- `--model=gpt‑4‑turbo` (default) – use the latest token‑rich model for higher fidelity.  
- `--region=IR` – forces the regional data layer to query kalatakco.com.  
- `--exclude-tokens` – list any additional Persian tokens you want to blacklist.
